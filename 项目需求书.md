### 5. 技术栈 (Technology Stack)

*   **HTML5:** 结构与语义化。
*   **CSS3:** 高级样式，如 `transform`, `transition`, `animation` 实现 Aceternity 风格的动效。使用 CSS 变量进行主题管理。考虑使用 PostCSS 进行样式优化和兼容性处理。
*   **JavaScript (Vanilla JS + GSAP (可选，但推荐)):**
    *   **Vanilla JS:** 核心交互逻辑、表单处理、响应式导航。
    *   **GSAP (GreenSock Animation Platform):** 若要实现更复杂、流畅且高性能的 Aceternity 风格动画，GSAP 是一个极佳的选择，它在兼容性和性能上远超原生CSS动画。可以用于视差滚动、元素入场动画、光效变化等。
    *   **Intersection Observer API:** 实现元素进入视口时的加载和动画效果（懒加载、渐显动画）。
*   **Lucide Icons:** SVG 引入或作为 Web Component 引入，确保最佳渲染效果和可定制性。
*   **CDN:** 关键资源（字体、GSAP库）通过 CDN 加速。

### 6. GSC 和 GA4 统计代码 (Analytics Integration)

*   **Google Search Console (GSC):**
    *   在 `<head>` 部分添加 GSC 验证的 `meta` 标签或通过 DNS 验证。
*   **Google Analytics 4 (GA4):**
    *   在 `<head>` 部分添加 GA4 的全局站点标签 (gtag.js) 代码。
    *   除了基础的页面浏览量，还需配置**事件跟踪**，重点监控：
        *   CTA 按钮点击 (特别是“免费试用”按钮)
        *   视频播放次数/进度 (如果嵌入演示视频)
        *   关键区段滚动深度 (衡量用户对内容的兴趣)
        *   表单提交成功事件

    ```html
    <!-- Google tag (gtag.js) -->
    <script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
    <script>
      window.dataLayer = window.dataLayer || [];
      function gtag(){dataLayer.push(arguments);}
      gtag('js', new Date());

      gtag('config', 'G-XXXXXXXXXX'); // 替换为你的GA4测量ID

      // 示例：追踪CTA按钮点击事件
      document.addEventListener('DOMContentLoaded', function() {
          const ctaButtons = document.querySelectorAll('.cta-button'); // 假设CTA按钮都有'cta-button'类
          ctaButtons.forEach(button => {
              button.addEventListener('click', function() {
                  gtag('event', 'cta_click', {
                      'event_category': 'Engagement',
                      'event_label': button.textContent,
                      'value': 1
                  });
              });
          });
      });
    </script>
    ```

### 8. 性能要求 (Performance Requirements)

*   **核心网络指标 (Core Web Vitals) 达标:**
    *   **LCP (Largest Contentful Paint):** 目标 < 2.5 秒。
    *   **FID (First Input Delay):** 目标 < 100 毫秒。
    *   **CLS (Cumulative Layout Shift):** 目标 < 0.1。
*   **优化资源加载:**
    *   **代码精简:** HTML、CSS、JS 文件进行彻底的压缩和去除不必要的空格/注释。
    *   **关键 CSS 内联:** 首屏所需的 CSS 直接内联到 HTML 中，消除渲染阻塞。
    *   **JS 延迟加载/异步加载:** 非首屏或不影响初始渲染的 JavaScript 文件使用 `defer` 或 `async` 属性。
    *   **图片懒加载:** 所有非首屏图片使用 `loading="lazy"`。
    *   **字体优化:** 使用 `font-display: swap`，仅加载所需字体子集，避免加载整个字体文件。
*   **CDN 策略:** 优先使用 CDN 提供所有静态资源，特别是大型 JS 库 (如 GSAP) 和图片。
*   **服务器配置:** 确保服务器启用 Gzip/Brotli 压缩，并配置浏览器缓存策略 (`Cache-Control`)。
*   **WebP 格式:** 所有位图图像优先采用 WebP 格式，并提供 JPG/PNG 回退。
*   **避免大量 DOM 节点:** 保持 HTML 结构简洁，避免不必要的嵌套。
